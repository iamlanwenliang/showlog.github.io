选择器

选择器用于我们给HTML元素添加样式，和在JavaScript中我们用来操作dom,下面就梳理下我们常用的选择器。

基本选择器

（1）类选择器（.class），匹配class包含（不是等于，是包含）特定类的元素。
（2）id选择器（#id），匹配特定id的元素。
（3）通用元素选择器（*），匹配页面的任何元素，这个我们基本上在项目开发中很少使用。
（4）元素选择器（element），选择匹配HTML元素。

组合选择器

（1）多元素选择器（A,B），同时匹配元素A和元素B。
（2）后代选择器（A B），匹配A元素的所有后代元素B，后代不只是子元素，还有子元素向下递归。
（3）子元素选择器（A>B）,匹配A元素的所有直接子元素。
（4）直接相邻选择器（A+B），匹配A元素之后的相邻的同级元素B。
（5）普通相邻选择器（A~B）匹配A元素之后的同级元素B（不管是不是直接相邻都可以）。

属性选择器

（1）[attribute],例如[name],就是选择所有带name属性的元素。
（2）[attribute = value],例如：[name = test],就是选择所有使用name="test"的元素。
（3）[attribute ~= value],例如：[name ~= test],就是选择name属性包含单词"test"的所有元素。
（4）[attribute ^= value],例如：[name ^= test],就是选择一个name属性的起始值等于"test"的所有元素。
（5）[attribute $= value],例如：[name $= test],就是选择一个name属性的结尾值等于"test"的所有元素。
（6）[attribute *= value],例如：[name *= test],就是选择一个name属性的包含"test"的所有元素。

伪类选择器

（1）:link,例如：a:link,就是选择所有未访问链接。
（2）:visited,例如：a:visited,选择所有访问过的链接。
（3）:hover,例如：a:hover,鼠标停留在链接上的状态。
（4）:active,例如：a:active,选择激活中的链接。
（5）:focus,例如：input:focus,选择元素输入后具有的焦点。
（6）:first-child，例如p:first-child,选择p元素的第一个子元素。
（7）:first-of-type,例如p:first-of-type,选择每个父元素是p元素的第一个p子元素。
（8）:last-child,例如p:last-child,选择所有p元素的最后一个子元素。
（9）:last-of-type,例如p:last-of-type,选择每个p元素是其母元素的最后一个p元素。
（10）:not(selector),例如:not(p),选择所有p以外的元素。
（11）:nth-child(n),例如p:nth-child(2),选择所有p元素的第二个子元素。
（12）:nth-last-child(n),例如p:nth-last-child(2), 选择所有p元素倒数的第二个子元素。
（13）:nth-last-of-type(n),例如p:nth-last-of-type(2),选择所有p元素倒数的第二个为p的子元素。
（14）:nth-of-type(n),例如p:nth-of-type(2),选择所有p元素第二个为p的子元素。
（15）:only-of-type,例如 p:only-of-type,选择所有仅有一个子元素为p的元素。
（16）:only-child,例如p:only-child,选择所有仅有一个子元素的p元素。
（17）:target,例如#news:target,选择当前活动#news元素(点击URL包含锚的名字)。
（18）:disabled,例如input:disabled,选择所有禁用的表单元素。
（19）:enabled,例如input:enabled, 选择没有设置 disabled 属性的表单元素。
（20）:in-range,例如input:in-range,选择在指定区域内的元素。
（21）:out-of-range,例如input:out-of-range,选择不在指定区域内的元素。
（22）:valid,例如input:valid,选择条件验证正确的表单元素。
（23）:invalid,例如input:invalid,选择条件验证错误的表单元素。
（24）:required,例如input:required,选择设置 required 属性的表单元素。
（25）:optional,例如input:optional,选择没有 required 属性，即设置 optional 属性的表单元素。
（26）:read-only,例如input:read-only,选择设置 readonly 只读属性的元素。
（27）:read-write,例如input:read-write,选择处于编辑状态的元素；input，textarea 和设置 contenteditable 的 HTML 元素获取焦点时即处于编辑状态。
（28）:checked,例如input:checked，匹配被选中的input元素，input元素包括radio和Checkbox。
（29）:empty,例如p:empty,匹配所有没有子元素的p元素。
（30）:default,匹配默认选中的元素(提交按钮总是表单的默认按钮)。
（31）:indeterminate,当某组中的单选框或复选框还没有选取状态时，:indeterminate 匹配该组中所有的单选框或复选框。
（32）:scope,匹配处于 style 作用域下的元素。当 style 没有设置 scope 属性时，style 内的样式会对整个 html 起作用。(试验阶段，注意兼容)
（33）:lang(language),匹配设置了特定语言的元素。设置特定语言可以通过为了 HTML 元素设置 lang=”” 属性，设置 meta 元素的 charset=”” 属性，或者是在 http 头部上设置语言属性。实际上，lang=”” 属性不只可以在 html 标签上设置，也可以在其他的元素上设置。
（34）:dir,匹配指定阅读方向的元素。当 HTML 元素中设置了 dir 属性时该伪类才能生效。现时支持的阅读方向有两种：ltr（从左往右）和 rtl（从右往左）。目前，只有火狐浏览器支持 :dir 伪类，并在火狐浏览器中使用时需要添加前缀( -moz-dir() )(试验阶段，注意兼容)。
（35）:root,匹配文档的根元素。一般的 html 文件的根元素是 html 元素。
（36）:fullscreen,匹配处于全屏模式下的元素。全屏模式不是通过按 F11 来打开的全屏模式，而是通过 Javascript 的 Fullscreen API 来打开的，不同的浏览器有不同的 Fullscreen API。目前，:fullscreen 需要添加前缀才能使用。

注：CSS 引入伪类和伪元素的概念是为了格式化文档树以外的信息。也就是说，伪类和伪元素是用来修饰不在文档树中的部分，比如，一句话中的第一个字母，或者是列表中的第一个元素。

伪元素选择器

（1）::before/:before,在被选元素前插入内容，需要使用content属性来指定要插入的内容，被插入的内容实际上不在文档树中。
（2）::after/:after,在被选元素后插入内容，特性用法同上。
（3）::first-letter/:first-letter,匹配元素中文本的首字母，被修饰的首字母不在文档书里面。
（4）::first-line/:first-line,匹配元素中第一行的文本，这个伪元素只能用在块元素中，不能使用在内联元素中。
（5）::selection,匹配被用户选中或者处于高亮状态的部分，在做兼容时加前缀，在火狐时添加-moz前缀。
（6）::placeholder,匹配占位符的文本，只有元素设置了placeholder属性的时候，该伪元素才能生效，伪元素不是css标准，它的实现可能在将来会有所改变，所以使用需谨慎，在一些版本（IE10，ff18及其以下）会使用单冒号的形式。
（7）::backdrop,用于改变全屏模式下的背景颜色，全屏模式的默认颜色为黑色。


注：使用时请注意支持性！
