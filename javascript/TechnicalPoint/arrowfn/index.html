ES6时代的箭头函数

前言

箭头函数是ES6新增的特性之一，它为JS这门语言提供了一种全新的书写函数的语法，程序员很懒的，我们热衷于各种脚本语言，语法糖。不就是因为他们能够让开发更有效率，让代码写的更少，更爽吗？箭头函数简化了函数的语法。

语法

基本语法：
(参数1, 参数2, …, 参数N) => { 函数声明 }
(参数1, 参数2, …, 参数N) => 表达式（单一）
//相当于：(参数1, 参数2, …, 参数N) =>{ return 表达式; }
// 当只有一个参数时，圆括号是可选的：
(单一参数) => {函数声明}
单一参数 => {函数声明}
// 没有参数的函数应该写成一对圆括号。
() => {函数声明}
高级语法：
//加括号的函数体返回对象字面表达式：
参数=> ({foo: bar})
//支持剩余参数和默认参数
(参数1, 参数2, ...rest) => {函数声明}
(参数1 = 默认值1,参数2, …, 参数N = 默认值N) => {函数声明}
//同样支持参数列表解构
let f = ([a, b] = [1, 2], {x: c} = {x: a + b}) => a + b + c;
f(); // 6

也就是说在写箭头函数的时候，我们记住一个顺序，参数、箭头、函数体、记住这个顺序记住就足够了，参数、箭头、函数体、这三个是必须的，函数名可以没有，但这三项必须有，一些简写的方式也是简写这三项里的东西。下面说下简写方式：

（1）只有一个参数时，() 可省略,例如：
//不简写
let fn = (a) =>{
console.log(a);
}
//简写
let fn = a =>{
console.log(a);
}

(2)当函数体只有一句的时候，函数的{}可以省略。
//不简写
let fn = (a) =>{
console.log(a);
}
//简写
let fn = (a) => console.log(a);

(3)函数体只有一条返回语句时，{} 和 return 都可以省略。
//不简写
let fn = (a) =>{
return a;
}
//简写
let fn = (a) => a ;
或者
let fn = (a) => return a;

(4)箭头函数 如果要返回一个对象要简写的话， 需要用()包住.
//不简写
let fn = () =>{
return {a:1};
}
//简写
let fn = () => ({a:1}) ;

注意：
箭头函数放 参数 的地方就在 () 内，
没有参数，() 必须写，
一个参数，() 可写可不写，
多个参数，() 必须写。
箭头函数放 函数体 的地方在 {}内，
函数体 就 一句 {} 可写可不写，
函数体 不止一句，{} 必须写。
如果不知道，() {} 写不写，该不该省略，那就写，写了不会错。

以下情况不能使用箭头函数

1.作为对象的方法。(本人不建议使用，至于为什么，只能说跟this有关系)
2.不能作为构造函数（箭头函数不能被new）
由于箭头函数的this不绑定的特点，所以不能使用箭头函数作为构造函数，实际上如果这样做了，也会报错。
3.定义原型方法

个人认为，箭头函数几乎可以完全取代function，除非是函数需要进行递归或者需要可变的this对象，其他场景我认为都可以使用箭头函数。
