JavaScript中const、let、var之间的区别

最近在使用ts编写代码的时候偶然混用了一下var声明变量，导致代码出现了异常，所以来研究下let、var和const的区别和适用场景。

var

1.var定义的变量可以修改，如果不初始化会输出undefined，不会报错.
2.申明了一个要么是全局，要么是函数级的变量，在JS函数中的var声明，其作用域是函数体的全部.
3.变量声明无论出现在代码的任何位置，都会在任何代码执行之前处理，也就是会被变量提升.

例如：
console.log(a)//undefined
var a ;
console.log(a)//undefined
function fn(){
   var b = 1;
   a = 22;
   console.log(b);//1
}
fn();
console.log(a);//22
console.log(b);// b is not defined

let

1.申明了一个块级域的局部变量，并且可以给它一个初始化值.
2.let声明的全局变量不是全局对象的属性.
3.形如for (let x...)的循环在每次迭代时都为x创建新的绑定。
4.用let重定义变量会抛出一个语法错误（SyntaxError）。

例如：
function fn(){
   var a = 0;
   {
     let b = 2;
     for(var i=0;i<3;i++){
       console.log(i);//0,1,2
       console.log(b);//2,2,2
     }
     console.log(b);//2
     console.log(a);//0
     let b = 2;//Identifier 'b' has already been declared
   }
   console.log(b);//b is not defined
   console.log(i);//3
}
fn();

const

1.在全局作用域或者函数内申明常量，但是必须初始化常量,定义的变量不可以修改；
2.常量不能和它所在作用域内其它变量或者函数拥有相同名称；

例如：
const a ;//Missing initializer in const declaration
const a = 0;
a = 1;//Assignment to constant variable.

以上就是三种声明的使用用法规则，适用场景各取所好！
